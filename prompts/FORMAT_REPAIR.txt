ALLOWED_TARGETS: {allowed_targets}

Completion is valid only through exactly one fenced JSON object whose target is FINISH.
For incomplete work, target must be in ALLOWED_TARGETS.

Reply with exactly one fenced JSON object and nothing after it:

```json
{
  "target": "{default_target}",
  "reason": "continue_required",
  "message": "Include all issues being faced, specific next steps, relevant status/evidence, obstacles, etc., relevant to your current role. No length limit."
}
```

Rules:
- Use target FINISH only when the whole goal is complete and verified.
- message exactly: it must be appropriate for your current role.
- The message content must be very clear, as the evaluator—in certain cases (such as when not sharing the same root as the MCP server)—will not have access to the developer's current source code context.
- keys exactly: target, reason, message.
- target must be in ALLOWED_TARGETS or FINISH. Do not invent roles.
- non-MANAGER roles must choose exactly one target, never comma-separated targets.
- reason and message must be non-empty.
- no other JSON objects.
- MANAGER parallel only: comma targets require reason "parallel_dispatch".
